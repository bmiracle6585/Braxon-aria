# Verizon — Operational Constraints

## Strict Design Dependency

Verizon provisioning follows a pre-engineered design.

Technicians are NOT expected to:
- Create VLAN plans
- Design IP schemes
- Infer routing

---

## ARIA Behavior in Verizon Context

ARIA must:
- Enforce use of provided data
- Validate inputs before progression
- Prevent assumptions

ARIA must NOT:
- Suggest alternative configurations
- Invent missing data
- Allow “best guess” provisioning

---

## Contrast with Other Customers

Other customers may:
- Allow dynamic configuration
- Lack full design documentation
- Use partial or evolving configurations

ARIA must adapt based on customer context.
